# AiTool 桌面工具

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![Platform](https://img.shields.io/badge/platform-Windows-orange.svg)]()
[![Python](https://img.shields.io/badge/Python-3.10%2B-blue)]()

AiTool 是一个 Windows 桌面效率工具，用于临时收纳文件和文件夹、快速打开或定位资源，并通过快捷动作执行脚本、应用、网页、文件夹复制和 SVN 操作。程序支持拖放和剪贴板输入，也可以隐藏到系统托盘，在需要时恢复窗口。

## 功能概览

- **拖放与粘贴**：将网址、文件、文件夹、脚本或应用拖入主窗口。`Ctrl+V` 可粘贴网址或路径。
- **文件中转站**：暂存文件和文件夹，支持双击打开、右键在资源管理器中定位，以及拖出复制。
- **快捷动作**：添加、编辑、删除和执行动作卡片；支持文件夹覆盖复制、启动 `.bat`/`.cmd`/`.py` 脚本、SVN 更新与提交、打开网页和启动应用。
- **窗口与主题**：主窗口支持亮色/暗色主题和置顶切换；拖到屏幕上边缘可以停靠并自动收起，移入收起后的横条可恢复。

## 使用说明

### 输入方式

1. **拖放**：在主窗口可见时，把网址、普通文件、文件夹、脚本或应用拖到窗口中。
   - 网址会生成打开网页的快捷动作。
   - `.exe` 和 `.lnk` 会生成启动应用的快捷动作。
   - `.bat`、`.cmd` 和 `.py` 会生成脚本动作。
   - 普通文件和文件夹会加入文件中转站。
   - 同时拖入多个目录时，可以识别为多路径覆盖复制动作。
2. **粘贴**：复制网址或文本形式的文件路径后，在窗口内按 `Ctrl+V`。粘贴内容使用与拖放相同的识别逻辑；托盘隐藏时请先从托盘恢复主窗口。

### 文件中转站

- 双击条目：使用 Windows 关联程序打开文件或文件夹。
- 右键条目：打开资源管理器并定位/选中对应路径；如果原文件已失效但父目录仍存在，则打开父目录。
- 从条目的名称或图标拖出：将路径作为文件拖放数据提供给资源管理器或其他支持文件拖放的程序，实现复制。
- 中转站内容会持久化保存；路径失效时条目会标记为“失效”，可通过条目右侧按钮移除。

### 快捷动作

- 点击动作卡片执行动作，点击卡片右侧箭头编辑动作。
- 使用“添加”按钮创建自定义动作，或编辑、删除已有动作。
- 当前动作类型包括：文件夹覆盖复制、启动脚本、SVN 更新、SVN 提交、打开网页和启动应用。
- 文件夹覆盖复制支持配置来源目录和目标目录；SVN 动作需要有效的工作副本路径，并使用系统中的 `svn.exe` 或 TortoiseSVN 客户端。

### 窗口行为

- 主窗口采用固定尺寸策略，不能通过普通窗口边框自由调整；窗口内容底部、状态栏上方的内部纵向拖拽区域可以调整窗口高度。
- 调整后的窗口位置和高度会保存，下一次启动时优先恢复；如果显示器或工作区发生变化，程序会重新选择可用位置。
- 将窗口拖到屏幕上边缘并停留会进入顶部停靠状态。窗口收起后保留约 `30px` 的横条，移入横条即可展开。顶部停靠与隐藏到系统托盘是两种独立的窗口状态。
- 主窗口右上角使用 Windows 原生标题栏关闭按钮。点击该按钮会隐藏主窗口并保留在系统托盘，不会直接退出程序。

### 托盘与设置

右键或打开系统托盘中的 AiTool 图标，可以使用以下菜单：

- **打开 AiTool**：恢复主窗口。
- **设置中心**：打开设置窗口。
- **开机自启动**：切换 Windows 开机自启动状态。
- **彻底退出**：停止托盘驻留并退出程序。

窗口内还可以切换亮色/暗色主题和窗口置顶状态。若只是要关闭窗口但继续保留后台运行，请使用右上角原生关闭按钮；需要结束进程时使用托盘菜单中的“彻底退出”。

### 数据位置

程序运行模式决定用户数据目录：

- **源码运行**：配置保存在项目的 `data\` 目录，例如 `document_station_entries.json`、`custom_modules.json`、`quick_actions.json` 和 `window_geometry.json`。
- **打包后的 EXE（frozen 模式）**：用户配置保存在 `%APPDATA%\AiTool\`。打包文件中的 `data\` 只作为内置默认配置来源，首次运行时会复制到该用户目录。

因此，更新或替换 EXE 不会覆盖 `%APPDATA%\AiTool\` 中已有的用户配置。若需要重置打包版配置，应在程序退出后备份并按需删除该目录，再重新启动程序。

## 快速开始

### 源码运行

需要 Python 3.10 或更高版本。在项目根目录安装依赖：

```powershell
pip install customtkinter pillow pystray tkinterdnd2 pyinstaller
```

运行程序：

```powershell
python run_desktop_tool.py
```

### PyInstaller 打包

项目规定 `AiTool桌面工具.spec` 为唯一打包入口。按项目当前配置使用以下命令：

```powershell
py -m PyInstaller "AiTool桌面工具.spec" --noconfirm --upx-dir "C:\Users\Administrator\AppData\Local\Programs\Python\Python312"
```

打包后的程序使用 `%APPDATA%\AiTool\` 保存用户配置，详见上面的“数据位置”。

## 测试

在项目根目录运行：

```powershell
python -m pytest tests/ -v
```

## 故障排查与退出方式

- **拖放没有反应**：确认主窗口当前可见，并将对象直接拖到窗口内容区域；窗口隐藏到托盘时不能接收拖放。确认已安装 `tkinterdnd2`，必要时重启程序后再试。
- **粘贴没有反应**：`Ctrl+V` 读取剪贴板中的文本网址或路径；请确认复制的是完整网址或有效路径，并且主窗口已经恢复显示。
- **脚本、应用或 SVN 动作失败**：检查动作中的路径是否存在、路径是否为绝对路径，并确认系统已安装相应的脚本运行环境、应用或 `svn.exe`/TortoiseSVN 客户端。
- **彻底退出**：点击主窗口右上角关闭按钮只会隐藏到托盘；请打开托盘菜单并选择“彻底退出”。
