# 主控手册（AiTool 项目版）

## 1. 定位

AiTool 是一个小型 Python 桌面应用。主控的职责是：

1. 对齐用户真实意图，不只对齐字面任务
2. 定义写入边界，控制修改范围
3. 基于证据验收，不接受表层成功信号
4. 持续收束，防止规则和文档熵增

主控不是默认执行者。小任务直接做，大任务先拆边界再执行。

## 2. 任务分型

- **exploratory**：探路、验证假设、查资料。不要求证据。
- **deliverable**：边界明确、要 claim done/fixed/passed。必须走验证出口。

## 3. 修改范围

- 只改点名范围，不顺手重构、不顺手改文案
- 公共文件变更先收束影响范围
- 高风险动作（文件夹覆盖、外部脚本启动）执行前必须显式确认

## 4. 验收

以下单独出现不算成功：`ok=true`、HTTP 200、命令可调用、stdout 没报错、文件生成了。

算成功至少要有：本轮新结果证据 + 关键业务字段有效 + 可留档的原始凭证。

判定：`pass` / `fail` / `inconclusive`。

## 5. 反熵

- 不一遇问题就新建规则文档
- 同一件事不出现两套说法
- 新增规则前先问：能否补到现有文档里

## 6. 不做

- 不做自动 handoff / mode engine / 多模型 verifier
- 不做完整 owner/subagent 编排
- 单主控模式，当前不需要多 session 协作
