# AiTool 项目定位说明

## 1. 项目身份

AiTool 是一个正式项目，同时也是一个 harness 实践项目。

它对真实需求承接、推进和交付负责，不再把自己定义成外部实验容器，也不再把“同步某个母仓”当成项目主叙事。

## 2. 项目承担什么

AiTool 当前承担三类事情：

1. 承接真实需求并推进交付
2. 用主控、执行、证据、治理结构控制风险与收口
3. 把在真实任务中证明有效的做法沉淀为可复用实践

## 3. 项目不再怎样定义自己

AiTool 不再把自己定义为：
- 外部实验沙箱
- 纯粹的结构同步容器
- 只围绕空项目部署展开的试跑项目

未来即使需要把稳定做法整理成文件包、skill 或插件，那也属于成果沉淀，不改变 AiTool 本身作为正式项目的身份。

## 4. harness 在本项目里的角色

harness 在 AiTool 里的角色是：
- 帮助主控识别阶段、边界和风险
- 给执行层提供护栏
- 给交付层提供验证出口
- 给治理层提供纠偏结构

它是项目的工程操作结构，不是项目身份本身。

## 5. 当前边界

当前阶段明确不做：
- 先于真实需求验证的完整平台化扩展
- 过早把“可扩展模块”做成完整插件平台
- 自动 handoff、retro、mode engine、多模型 verifier 等 Phase 2 结构

## 6. 背景说明

本项目早期确实经历过一轮结构收口与同步工作，但那只是历史背景。
当前对外口径只保留一个定义：

> AiTool 是按正式项目标准推进真实需求，并在交付中沉淀 harness 实践的项目。
